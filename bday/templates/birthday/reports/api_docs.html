Daily Report API

Integration details for your external report collector

This endpoint returns the daily outreach report in JSON format so another application can collect sent email activity, SMS activity, failures, and per-patient details.

GET {{ api_url }}
Access Notes
Authentication is required on every request.
Headers supported
`Authorization: Bearer <token>`
`X-API-Key: <token>`
Endpoint
Basic request contract
Method
GET
Path
{{ api_path }}
Response
JSON
Auth
Bearer token or X-API-Key
Query Parameters
Parameter Required Description
date No Report date in YYYY-MM-DD format. Defaults to today.
patient_ids No Comma-separated patient IDs to restrict the report, for example 1,2,3.
Authentication
Server-side token configuration

Configure one of these environment variables on the Django app server: 

REPORT_API_TOKEN=your-secret-token
DAILY_REPORT_API_TOKEN=your-secret-token

The endpoint accepts either variable. If both exist, REPORT_API_TOKEN is used first.

Request Examples
Ready-to-use examples for your second app
Get today's report
curl -H "X-API-Key: your-secret-token" "{{ api_url }}"
Get a report for a specific date
curl -H "Authorization: Bearer your-secret-token" "{{ api_url }}?date={{ sample_date }}"
Get a filtered report for selected patients
curl -H "X-API-Key: your-secret-token" "{{ api_url }}?date={{ sample_date }}&patient_ids=12,18,44"

Error Responses
Common integration failures
401 Unauthorized
{"status":"error","message":"Unauthorized daily report request."}
400 Invalid Date
{"status":"error","message":"Invalid date. Use YYYY-MM-DD."}
503 Missing Server Token
{"status":"error","message":"Daily report API token is not configured."}
Field Notes
What the consuming app should expect
  • birthdays_total, birthdays_wished, and birthdays_unwished give the exact birthday counts for the selected day.
  • birthdays.details returns the full patient list for birthdays on that date, including whether each patient was already wished.
  • activities only includes Email Sent and SMS Sent entries for the selected day.
  • failures includes scheduled wishes with status Failed updated on the selected day.
  • created_at, scheduled_for, and updated_at are ISO-8601 timestamps.
  • practice may be null when a patient is not linked to a practice.
